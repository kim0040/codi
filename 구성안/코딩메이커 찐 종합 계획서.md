
# 코딩메이커 아카데미 허브 (CodingMaker Academy Hub)

# 종합 기획 및 기술 명세서 (v1.0)

## 1. 프로젝트 개요

### 1.1. 프로젝트명

코딩메이커 아카데미 허브 (CodingMaker Academy Hub)

### 1.2. 프로젝트 비전

단순한 학원 홈페이지를 넘어, **학습 관리(LMS), 출결, 학생/학부모 커뮤니케이션, 프로젝트 협업**이 모두 하나의 공간에서 유기적으로 이루어지는 '올인원(All-in-One) 디지털 허브'를 구축합니다.

### 1.3. 프로젝트 목표

1. **통합 LMS 구축:** 관리자가 '클래스(수업)'를 개설하고, 주차별 커리큘럼(강의 자료, 파일)을 업로드하며, 학생들의 수강 관리를 체계화합니다.
    
2. **자동화된 운영:** 키오스크(태블릿) 기반의 출석 시스템을 도입하여 출결 관리를 자동화하고, 이 데이터를 학부모가 실시간으로 확인할 수 있게 합니다.
    
3. **강화된 커뮤니티:** 코딩 학원의 특수성을 살린 '코드 하이라이팅', '프로젝트 협업' 기능을 탑재하여, 학생들 간의 지식 공유와 협업을 촉진합니다.
    
4. **확장성 및 안정성:** 일일 200명 이상의 동시 접속자를 감당할 수 있도록, 기존 테스트 프로젝트('Compile-Room')의 성능 및 보안 문제를 해결한 안정적인 아키텍처를 수립합니다.
    
5. **편리한 소통:** 게시판, Q&A, 실시간 채팅, 포괄적인 알림 시스템을 통해 학생, 교사, 학부모 간의 소통을 원활하게 합니다.
    

### 1.4. 핵심 타겟 사용자

- **최고 관리자 (T1):** 학원 원장. 사이트의 모든 기능을 제어합니다.
    
- **관리자 (T2):** 학원 교사. 학생 관리, 클래스 운영, 콘텐츠 작성을 담당합니다.
    
- **정회원 (T3):** 학원 재원생. LMS, 커뮤니티, 프로젝트의 핵심 사용자입니다.
    
- **학부모 (T3-Parent):** 재원생의 학부모. 자녀의 출석 및 학업 현황을 모니터링합니다.
    
- **준회원 (T4):** 회원가입 직후 또는 관심 있는 외부인. 제한된 정보에 접근합니다.
    
- **방문자 (T5):** 비로그인 사용자. 학원 홍보 페이지를 열람합니다.
    

## 2. 핵심 원칙 및 제약 조건

### 2.1. 100% 무료 라이선스

본 프로젝트에 사용되는 모든 기술 스택, 라이브러리, 인프라는 상업적 이용을 포함하여 **완전 무료** 플랜/라이선스를 기반으로 합니다. (예: MIT, Apache 2.0 및 Vercel, PlanetScale 등의 Free Tier)

### 2.2. 성능 및 확장성

일일 200명 이상의 트래픽을 원활하게 처리해야 합니다. 이를 위해 `SQLite`와 같은 경량 DB가 아닌, 대규모 트래픽 처리에 검증된 `MySQL` 호환 클라우드 데이터베이스를 사용하며, 서버는 트래픽에 따라 자동 확장되는 서버리스(Serverless) 아키텍처를 채택합니다.

// 첨언: 모든 건 학원 로컬 서버를 통해 운영할 예정임
### 2.3. 보안 및 프라이버시

1. **개인정보:** 모든 민감 정보(비밀번호, API 키)는 코드에서 분리되어 환경 변수로 관리됩니다. 비밀번호는 `bcrypt` 해싱을 통해 저장됩니다.
    
2. **채팅 프라이버시:** 모든 채팅 메시지는 DB 저장 시 어플리케이션 레벨에서 `대칭키 암호화`됩니다. 관리자(T1 포함)라도 참여하지 않은 채팅방의 메시지는 절대 열람할 수 없습니다.
    
3. **데이터 접근:** 모든 API는 `NextAuth.js`의 세션과 역할을 기반으로 엄격한 접근 제어(RBAC)를 수행합니다.
    

### 2.4. UI/UX

- `Compile-Room`의 미적 감각을 계승하되, 버그(예: 다크 모드)를 해결하고 일관성을 높입니다.
    
- `shadcn/ui`와 `tailwind-variants`를 활용하여 체계적인 디자인 시스템을 구축합니다.
    
- 모든 페이지는 모바일, 태블릿, 데스크탑을 완벽하게 지원하는 반응형으로 제작되어야 합니다.

## 4. 사용자 권한 (Role) 상세 정의

`NextAuth.js`의 `session` 콜백과 `Prisma`의 `User` 모델을 연동하여 역할을 관리합니다.

|   |   |   |   |
|---|---|---|---|
|**티어 (Tier)**|**역할명**|**권한 설명**|**주요 기능**|
|**Tier 1**|**최고 관리자 (Super Admin)**|(학원 원장) 사이트의 모든 기능을 제어.|사이트 전체 설정, DB 백업/롤백, 모든 사용자 정보/권한 할당, 출석(Kiosk) 계정 생성, 콘텐츠(소개, 클래스) 수정|
|**Tier 2**|**관리자 (Admin)**|(교사) 학생 관리 및 교육 콘텐츠 생성 권한.|학생 정보 관리, T3까지 권한 승급, 공지사항/명예의 전당 작성, Q&A 답변, **클래스 생성/관리**|
|**Tier 3**|**정회원 (Full Member)**|(재원생) 학원 커뮤니티 및 학습 기능의 핵심 사용자.|클래스 수강, 게시판/댓글 작성, 채팅방 생성/참여, **프로젝트 생성/참여**, 회원 목록 조회|
|**Tier 3 (특수)**|**학부모 (Parent)**|(보호자) 자녀의 학원 활동 모니터링.|계정 설정에서 학생 계정 인증을 통해 '학부모'로 전환 가능. T1/T2 관리자가 수동으로 연결/해제 가능. 연결된 '정회원'(자녀)의 출석 정보, 수강 클래스 목록만 열람.|
|**Tier 3 (특수)**|**명예회원 (Honor Member)**|(졸업생) 커뮤니티 활동 권한 유지.|'정회원'과 거의 동일한 커뮤니티 권한 (클래스 수강 등 일부 LMS 기능 제외)|
|**Tier 4**|**준회원 (Associate Member)**|(가입 직후/방문자) 제한된 커뮤니티 참여.|(이름 변경: 비정규회원 -> 준회원) 게시판 읽기, Q&A 질문 작성, 전체 채팅방 참여. (파일 업로드 불가)|
|**Tier 5**|**비로그인 (Guest)**|(방문자) 공개된 학원 정보만 열람.|학원 소개, 수강 과정, 공지사항, 명예의 전당 등 공개 페이지만 접근 가능.|
|**특수**|**출석용 계정 (Kiosk)**|(API 계정) 출석 API 호출 전용.|세션 로그인이 아닌, 발급된 API 키 기반으로 출석 API 엔드포인트만 호출 가능.|

## 5. 핵심 기능 상세 명세 (Functional Specification)

### 5.1. 공통 및 콘텐츠

- **헤더:** 학원 로고, 학원 소개, **수강 과정** (클래스+비용), 공지사항, 자유게시판, 채팅, Q&A, **명예의 전당** (자랑거리), 로그인/대시보드, **알림 벨**.
    
- **홈페이지:** 학원 소개(온보딩), 학원 일정(캘린더), 인기/최신 게시글, '명예의 전당' 요약.
    
- **학원 소개 / 수강 과정:** T1(최고 관리자)이 수정 가능한 정적 콘텐츠 페이지. (Markdown 또는 Tiptap 에디터로 수정 기능 제공)
    
- **명예의 전당:** T2(관리자) 이상 작성. (학생 성과, 대회 수상, 프로젝트 결과물 아카이빙)
    

### 5.2. 커뮤니티 시스템

- **자유게시판 (T3+):**
    
    - `Tiptap` 에디터를 통한 글 작성/수정 (이미지, 파일, **코드 스니펫** 포함).
        
    - 게시글 삭제: 작성자 및 T2 이상 가능. 삭제 시 `isDeleted: true` 플래그 처리, 사유(`deleteReason`) 필드 추가. 리스트에는 "(삭제된 게시물입니다)"로 표시.
        
- **Q&A (T4+ 질문, T2+ 답변):**
    
    - T4 이상 질문 작성 가능. `isSecret: true` 플래그로 비밀글 관리 (T2 이상 및 작성자만 열람/답변 가능).
        
- **채팅 (T4+ 읽기/쓰기, T3+ 생성):**
    
    - T4는 '전체 채팅방'만 참여 가능. T3 이상은 채팅방 생성, 초대, 1:1 DM 가능.
        
    - **파일/코드 공유:** Cloudinary 및 '코드 하이라이팅' 기능 연동.
        
    - **메시지 암호화:** `Message.content` 필드는 `crypto` 모듈로 암호화되어 DB에 저장.
        
    - **관리자 삭제 (T1):** T1은 참여하지 않은 채팅방이라도 '강제 삭제' 가능. (단, 메시지 열람은 불가).
        
        - **삭제 절차:** 1) 삭제 버튼 클릭 -> 2) 경고 모달 표시 -> 3) 모달에 특정 문구 (예: `(채팅방 이름)은(는) 사이트에 위험한 채팅방이므로 이에 삭제합니다.`)를 사용자가 **직접 타이핑** -> 4) 일치 시 삭제 API 호출. (실수 방지)
            
    - **메시지 삭제:** 작성자는 자신 메시지 삭제 가능 (단, `(삭제된 메시지입니다)`로 표시).
        

### 5.3. 학습 관리 시스템 (LMS)

- **클래스 (Class):**
    
    - **관리 (T1/T2):** 클래스(수업) 생성 (수업명, 설명, 비용, **수업 요일** 설정). T3 회원 목록에서 검색하여 '수강생'으로 추가/삭제.
        
    - **수강 (T3):** 대시보드에서 '수강 중인 클래스' 목록 확인.
        
- **커리큘럼 (Curriculum):**
    
    - **관리 (T1/T2):** 클래스 내에 '주차별(Week)' 커리큘럼 생성.
        
    - 콘텐츠(텍스트, 파일, 예제 코드) 업로드.
        
    - `openDate` (공개 날짜)를 설정하여 특정 날짜/시간에 자동으로 학생들에게 공개(잠금 해제)되도록 설정.
        

### 5.4. 프로젝트 협업 시스템 (Git-Lite)

- **목표:** Git의 복잡성 없이, 학생들이 협업하고 산출물을 관리할 수 있는 가벼운 환경 제공.
    
- **권한:** T3 이상 (학부모 제외) 생성 및 참여 가능.
    
- **기능:**
    
    - **프로젝트 생성:** 이름, 설명, 멤버 초대(회원 리스트에서 검색).
        
    - **전용 채팅방:** 프로젝트 멤버만 참여하는 전용 채팅방 자동 생성.
        
    - **파일 관리:** 프로젝트 전용 Cloudinary 폴더에 파일 업로드.
        
    - **변경 로그 (`ProjectLog`):** "누가", "언제", "무엇을" 했는지(파일 업로드, 상태 변경) 자동으로 기록.
        
    - **버전 관리 (단순화):** '롤백' 대신 '이전 버전 파일 다운로드' 기능 제공. (예: `main_v1.py`, `main_v2.py`...)
        
    - **상태 관리:** 프로젝트 상태를 '진행 중', '완료' 등으로 설정.
        

### 5.5. 학원 운영 시스템

- **출석 시스템 (Kiosk):**
    
    - **계정 (T1):** T1이 '사이트 관리'에서 '출석용 계정' 생성 -> 암호화된 `apiKey` (UUID) 발급. T1이 이 키를 태블릿에 _직접_ 입력.
        
    - **태블릿 (Kiosk 기기):**
        
        1. 전용 로그인 페이지 (`/kiosk/login`)에서 `apiKey`를 입력.
            
        2. 로그인 성공 시, `apiKey`를 브라우저 `localStorage`에 저장하고, 즉시 `/kiosk/attendance` 페이지로 리다이렉트 (다른 페이지 접근 불가).
            
        3. 이 페이지는 10분 유휴 로그아웃이 적용되지 않음.
            
    - **출석 로직:**
        
        1. 학생이 자신의 `이름#태그` (예: `김현민#0001`) 입력 후 '확인'.
            
        2. `localStorage`의 `apiKey`와 `이름#태그`를 `POST /api/kiosk/check-in` 로 전송.
            
        3. **서버:** 1) `apiKey` 유효성 검증. 2) `이름#태그`로 `User` 조회. 3) 해당 `User`가 오늘 수강해야 할 `Class` (수업 요일 대조)가 있는지 확인. 4) `AttendanceLog`에 출석 기록 생성. 5) "OOO님 (XX 클래스) 출석 완료" 응답 전송.
            
    - **원격 로그아웃 (T1):** T1이 '사이트 관리'에서 Kiosk 계정의 `apiKey`를 '비활성화' 또는 '재생성'하여 태블릿을 원격으로 무효화.
        
- **학부모 연동:**
    
    - **셀프 인증:** T4(준회원)가 대시보드 '계정 설정'에서 '학부모이신가요?' 버튼 클릭 -> 자녀(T3)의 계정으로 로그인 인증 요청 -> 인증 성공 시, 해당 계정은 `PARENT` 역할로 자동 전환되고 `ParentLink`로 연결됨.
        
    - **수동 관리:** T1/T2 관리자가 수동으로 `User`를 `PARENT`로 지정하고 `ParentLink`를 생성/삭제할 수 있음.
        
    - **권한:** `PARENT` 역할은 대시보드에서 _오직_ 연결된 자녀의 '출석 정보'와 '수강 중인 클래스' 목록만 열람 가능.
        

### 5.6. 핵심 공유 서비스 (Cross-Cutting)

- **파일 업로드:**
    
    - 모든 파일(이미지, PDF, 코드)은 `POST /api/upload` 엔드포인트를 통해 Cloudinary로 전송됩니다.
        
    - API는 파일의 종류, 업로더, 위치(게시판, 프로젝트 등)에 따라 Cloudinary 내의 경로를 동적으로 지정합니다. (예: `uploads/boards/{postId}/...`, `uploads/projects/{projectId}/...`)
        
    - DB(`Post`, `Project` 등)에는 파일의 `url`과 `메타데이터(JSON)`만 저장됩니다.
        
- **코드 하이라이팅 (백준/Gist 기능):**
    
    1. **스니펫 (붙여넣기):** `Tiptap` 에디터에서 '코드 블록' 선택 -> 언어(Python, JS 등) 지정 -> 해당 내용은 Markdown 형식(예: ```python ... ```)으로 DB `content` 필드에 텍스트로 저장. -> 뷰(View) 페이지에서 `react-syntax-highlighter`가 이 텍스트를 감지하여 하이라이팅.
        
    2. **파일 (업로드):** `.py`, `.js` 등 코드 파일 업로드 -> Cloudinary에 저장 -> 게시물에 '미리보기' 버튼 생성 -> 클릭 시 모달 창 -> 서버가 Cloudinary에서 해당 파일의 원시 텍스트(raw text)를 `fetch` -> `react-syntax-highlighter`로 전달하여 모달 내부에 하이라이팅된 코드를 표시.
        
- **실시간 알림 시스템:**
    
    - 헤더에 '알림 벨' 아이콘 배치, 읽지 않은 알림이 있으면 배지 표시.
        
    - 알림 클릭 시 드롭다운으로 목록 표시 (메시지, 시간, 바로가기 링크).
        
    - `Notification` 모델을 사용하며, 아래 상황에서 **자동 생성**됩니다:
        
        - **커뮤니티:** 내 게시물에 '새 댓글', 내 Q&A에 '새 답변', 참여 중인 채팅방/DM에 '새 메시지'.
            
        - **LMS:** 내가 수강 중인 클래스에 '새 커리큘럼 주차 해금', 관리자가 나를 '클래스에 초대'.
            
        - **프로젝트:** 누군가 나를 '프로젝트에 초대', 참여 중인 프로젝트에 '새 파일' 또는 '새 로그'.
            
        - **계정:** 관리자가 내 역할을 '정회원으로 승급'.
            

## 6. 데이터베이스 스키마 (Prisma Schema - 컨셉)

```
// --- User & Auth ---
model User {
  id            String    @id @default(cuid())
  name          String?
  email         String?   @unique
  password      String?   // Hashed
  userTag       String    @unique @default(cuid()) // For Kiosk: e.g., #0001
  role          UserRole  @default(ASSOCIATE_MEMBER)
  
  // --- Relationships ---
  enrollments   StudentClassEnrollment[] // As Student
  parent        ParentLink?              @relation("StudentToParent")
  child         ParentLink?              @relation("ParentToStudent")
  attendanceLogs AttendanceLog[]
  projectMemberships ProjectMember[]
  notifications Notification[]
  
  // ... (Other relations: posts, comments, chatRooms, messages)
}

enum UserRole {
  SUPER_ADMIN      // T1
  ADMIN            // T2
  FULL_MEMBER      // T3
  HONOR_MEMBER     // T3-Honor
  PARENT           // T3-Parent
  ASSOCIATE_MEMBER // T4
}

model ParentLink {
  id         String @id @default(cuid())
  studentId  String @unique // Student User ID
  student    User   @relation("StudentToParent", fields: [studentId], references: [id])
  parentId   String @unique // Parent User ID
  parent     User   @relation("ParentToStudent", fields: [parentId], references: [id])
}

// --- LMS (Learning Management System) ---
model Class {
  id            String    @id @default(cuid())
  name          String
  description   String?
  cost          Int?
  classDays     Int[]     // [1, 3, 5] for Mon, Wed, Fri (0=Sun)
  
  enrollments   StudentClassEnrollment[]
  curriculums   CurriculumWeek[]
  attendanceLogs AttendanceLog[]
  chatRoomId    String?   @unique // Class-specific chat room
  chatRoom      ChatRoom? @relation(fields: [chatRoomId], references: [id])
}

model StudentClassEnrollment {
  id        String  @id @default(cuid())
  userId    String
  user      User    @relation(fields: [userId], references: [id], onDelete: Cascade)
  classId   String
  class     Class   @relation(fields: [classId], references: [id], onDelete: Cascade)
  @@unique([userId, classId])
}

model CurriculumWeek {
  id          String    @id @default(cuid())
  classId     String
  class       Class     @relation(fields: [classId], references: [id], onDelete: Cascade)
  weekNumber  Int
  title       String
  content     String?   // Markdown/HTML content (with code blocks)
  openDate    DateTime  // Date when this week becomes visible
  files       Json?     // [{ name: "file.pdf", url: "..." }]
}

// --- Operations ---
model AttendanceLog {
  id        String    @id @default(cuid())
  userId    String
  user      User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  classId   String
  class     Class     @relation(fields: [classId], references: [id], onDelete: Cascade)
  checkInTime DateTime  @default(now())
  status      AttendStatus @default(PRESENT)
  markedByKiosk Boolean @default(false)
  markedByAdminId String? 
}

enum AttendStatus { PRESENT, LATE, ABSENT }

model KioskAccount {
  id      String @id @default(cuid())
  name    String // e.g., "Main Entrance Tablet"
  apiKey  String @unique // Store a HASH of the API key
  isActive Boolean @default(true)
}

// --- Collaboration (Projects) ---
model Project {
  id          String    @id @default(cuid())
  name        String
  description String?
  status      ProjectStatus @default(IN_PROGRESS)
  members     ProjectMember[]
  logs        ProjectLog[]
  chatRoomId  String?   @unique // Project-specific chat room
  chatRoom    ChatRoom? @relation(fields: [chatRoomId], references: [id])
  files       Json?     // [{ name: "main_v2.py", url: "...", ver: 2, uploaderId: "..." }]
}

enum ProjectStatus { IN_PROGRESS, COMPLETED, ARCHIVED }

model ProjectMember {
  id        String  @id @default(cuid())
  userId    String
  user      User    @relation(fields: [userId], references: [id], onDelete: Cascade)
  projectId String
  project   Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  role      String  // e.g., "Owner", "Member"
  @@unique([userId, projectId])
}

model ProjectLog {
  id          String    @id @default(cuid())
  projectId   String
  project     Project   @relation(fields: [projectId], references: [id], onDelete: Cascade)
  userId      String    // Log generated by this user
  user        User      @relation(fields: [userId], references: [id])
  message     String    // e.g., "User A uploaded file 'main.py'"
  createdAt   DateTime  @default(now())
}

// --- Community (Modified) ---
model Post {
  // ... (id, title, createdAt, etc.)
  content     String    // Tiptap's Markdown/HTML output (with code blocks)
  isDeleted   Boolean   @default(false)
  deleteReason String?
  deletedById  String?
  // ... (relations to User, Comments, etc.)
}

model Message {
  // ... (id, createdAt, etc.)
  content     String    // [ENCRYPTED] Tiptap output (with code blocks)
  isDeleted   Boolean   @default(false)
  // ... (relations to User, ChatRoom)
}

// --- Core Services ---
model Notification {
  id        String   @id @default(cuid())
  userId    String   // The user who receives the notification
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  message   String   // "A님이 '프로젝트'에 새 파일을 올렸습니다."
  link      String   // Link to the content (e.foo., /posts/123, /project/abc)
  isRead    Boolean  @default(false)
  createdAt DateTime @default(now())
  @@index([userId])
}
```
